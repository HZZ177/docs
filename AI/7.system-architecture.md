**分层概要：**

| 层级 | 说明 |

**各模块职责：**

| 模块 | 功能 | 对接后端 |
|------|------|---------|
| **Agent 管理** | 定义 Agent（名称、模型、温度、Prompt），支持启停和复制 | REST API CRUD |
| **Skill 管理** | 上传 skill.md 自动解析预填，或页面手动创建技能（工具模式/脚本模式） | REST API CRUD + parse |
| **Scene 编排** | 核心页面：选主 Agent + 添加子 Agent，为每个 Agent 绑定工具和技能，配置定时任务开关 | REST API CRUD（一次性提交全量编排） |
| **MCP 配置** | 添加 MCP Server 地址，刷新获取远程工具列表，勾选导入，ping 检测在线状态 | REST API CRUD + refresh + import |
| **记忆配置** | MVP 不开放，使用框架默认值（场景记忆 + 用户画像均启用） | — |
| **调试面板** | 选择 Scene 进行对话调试，左侧对话区 + 右侧调试面板（调用链路 / LLM 详情 / 记忆状态） | WebSocket |

**五大模块职责总结：**

| 模块 | 核心职责 | 输入 | 输出 |
|------|---------|------|------|
| **① 管理模块** | 配置资源的增删改查、业务校验、引用关系维护 | 前端 REST 请求 | DB 记录 |
| **② 对话服务** | WebSocket 接入、会话生命周期管理、消息路由、Skill 手动触发拦截 | WS 消息 | 组装请求→执行引擎、WS 事件→前端 |
| **③ 执行引擎** | 从 DB 读配置 → 动态组装 Agent → 流式执行 → 事件处理推送 | scene_id + 用户消息 | 流式 token + 工具调用事件 |
| **④ 记忆系统** | 场景记忆召回/抽取、用户画像加载/更新、对话上下文持久化 | 用户查询/对话历史 | 记忆文本注入 Prompt |
| **⑤ 定时任务** | cron 调度 → 复用执行引擎 → 结果通知 | DB 中的任务定义 | 执行日志 + 通知消息 |

## 四、核心数据流转

### 4.1 配置流：管理页面 → DB → 执行引擎

管理员在前端配置的数据如何在运行时被引擎消费。

```
    管理员操作                        数据存储                      运行时消费
   ───────────                      ────────                    ──────────

  ┌─────────────┐               ┌──────────┐              ┌─────────────────┐
  │ 创建/编辑    │  REST API     │          │  每次请求    │                 │
  │ Agent       ├──────────────→│ agents   ├─────────────→│                 │
  │ (模型/Prompt)│               │          │  从 DB 读取  │                 │
  └─────────────┘               └──────────┘              │                 │
                                                           │   执行引擎      │
  ┌─────────────┐               ┌──────────┐              │   (assembler)   │
  │ 创建/编辑    │  REST API     │          │  最新配置    │                 │
  │ Skill       ├──────────────→│ skills   ├─────────────→│  动态组装       │
  │ (指令/脚本)  │               │          │              │  Agent 实例     │
  └─────────────┘               └──────────┘              │                 │
                                                           │  ● 热更新       │
  ┌─────────────┐               ┌──────────┐              │    无需重启     │
  │ Scene 编排   │  REST API     │ scenes + │              │    修改即生效   │
  │ (主从/绑定)  ├──────────────→│ bindings ├─────────────→│                 │
  └─────────────┘               └──────────┘              │                 │
                                                           │                 │
  ┌─────────────┐               ┌──────────┐              │                 │
  │ MCP Server  │  REST API     │mcp_servers│  刷新后     │                 │
  │ 刷新Schema  ├──────────────→│mcp_tools ├─────────────→│                 │
  └─────────────┘               └──────────┘              └─────────────────┘

  关键设计：Agent 实例是请求级别的（每次对话创建新实例），不是应用级单例。
           assembler 每次都从 DB 读取最新配置，天然支持热更新。
           唯一做缓存的是 LLM 实例（创建开销大，配置变化时自动重建）。
```

### 4.2 对话流：用户消息 → Agent 执行 → 流式响应

一次完整对话请求的端到端流转。

```
  三方系统/调试面板                     后端服务                        外部依赖
  ─────────────                     ────────                        ────────

  ┌──────────┐    WebSocket     ┌───────────┐
  │          │  create_session  │           │
  │ 用户/    ├─────────────────→│ 对话服务   │
  │ 调试面板  │                  │           │
  │          │    chat 消息     │ ① 设置请求上下文
  │          ├─────────────────→│   (user_id + extra_params + token?)
  │          │                  │
  │          │                  │ ② 检查 "/" 前缀 → Skill 手动触发？
  │          │                  │   是 → 拦截处理（不经 LLM 判断）
  │          │                  │   否 → 正常对话流程 ↓
  │          │                  │
  │          │                  │ ③ 调用执行引擎
  │          │                  └──────┬────┘
  │          │                         │
  │          │                         ▼
  │          │                  ┌──────────────┐
  │          │                  │              │     ┌────────────┐
  │          │                  │  执行引擎     │────→│ SeekDB     │
  │          │                  │              │ 读   │ (配置/会话) │
  │          │                  │ ④ 从DB加载    │ 配置 └────────────┘
  │          │                  │   Scene配置   │
  │          │                  │              │     ┌────────────┐
  │          │                  │ ⑤ 组装Agent   │←───│ 记忆系统    │
  │          │                  │   注入记忆     │ 召回 │ (PowerMem) │
  │          │                  │              │ 画像 └────────────┘
  │          │                  │ ⑥ Agent执行   │
  │          │                  │   ReAct循环   │     ┌────────────┐
  │          │                  │              │────→│ LLM 网关    │
  │          │                  │   LLM推理     │ 推理 │ (大模型)    │
  │          │  ◀── stream ──── │              │     └────────────┘
  │          │  ◀── tool_start──│   工具调用?   │
  │          │  ◀── tool_end ── │   ├ 内置工具  │     ┌────────────┐
  │          │                  │   ├ MCP工具 ──┼────→│ MCP Server │
  │ ◀ 流式   │  ◀── subagent ──│   └ task工具  │ 透传 │ (业务工具)  │
  │   推送   │      _start/end │     (SubAgent │ 认证 └────────────┘
  │          │                  │      自动调度) │
  │          │  ◀── completed ──│              │
  │          │                  │ ⑦ 事件处理    │
  │          │                  │   → WS推送    │
  │          │                  └──────────────┘
  │          │
  │          │  close_session   ┌──────────────┐
  │          ├─────────────────→│  对话服务     │
  │          │                  │              │     ┌────────────┐
  └──────────┘                  │ ⑧ 异步触发    │────→│ 记忆系统    │
                                │   记忆抽取    │     │ · 场景记忆  │
                                │   画像更新    │     │   抽取存储  │
                                │              │     │ · 用户画像  │
                                └──────────────┘     │   增量更新  │
                                                     └────────────┘
```

**对话流关键环节说明：**

| 环节 | 功能逻辑 |
|------|---------|
| ① 设置请求上下文 | 将 user_id、extra_params、token(可选) 存入 ContextVar，全链路可取 |
| ② Skill 手动触发 | "/" 前缀消息被框架拦截，匹配 Skill name，校验权限后直接执行，不经 LLM |
| ③④ 配置加载 | assembler 从 DB 读取 Scene 全量配置（Agent 定义 + 绑定关系 + 记忆配置） |
| ⑤ 组装 Agent | 渲染 Prompt（模板变量替换）→ 注入记忆（画像 + 场景记忆）→ 注入 Skill 索引 → 创建 LangChain/DeepAgents 实例 |
| ⑥ Agent 执行 | ReAct 循环：LLM 推理 → 决定调用工具 → 执行工具 → 结果回注 → 继续推理，SubAgent 通过 task 工具由框架自动调度 |
| ⑦ 事件处理 | 将 Agent 执行产生的事件流（token/工具/SubAgent）转为 WebSocket 消息格式实时推送 |
| ⑧ 记忆抽取 | Session 关闭后异步触发：场景记忆抽取（事实提取+去重+存储）+ 用户画像增量更新 |

### 4.3 定时任务流：cron 触发 → 复用执行引擎 → 通知

  │            │                │    creator_*) │    └──────────┘
  └────────────┘                │              │

                                       │            └──────────────┘
                                       ▼
                                ┌──────────────┐
                                │  结果通知器   │
                                │              │
                                │ ⑤ 读取通知   │    ┌──────────┐
                                │   配置       │───→│ scenes表  │
                                │  (Scene的    │    │通知渠道   │
                                │   通知渠道)   │    └──────────┘
                                │              │
                                │ ⑥ 判断是否   │
                                │   需要通知    │
                                │  (Agent已在  │
                                │   task_prompt│
                                │   中嵌入条件) │
                                │              │
                                │ ⑦ 调用通知   │    ┌──────────┐
                                │   工具发送    │───→│ 通知渠道   │
                                │   给创建者   │    │(企业微信等)│
                                └──────────────┘    └──────────┘

  与正常对话的对比：
  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │  正常对话:  WS消息 → 对话服务 → 执行引擎 → WS推送给用户   │
  │  定时任务:  cron   → 任务分发 → 执行引擎 → 结果写DB+通知  │
  │                                                          │
  │  差异仅在触发入口和结果出口，Agent 执行链路完全复用         │
  │                                                          │
  └──────────────────────────────────────────────────────────┘
```

### 4.4 记忆流：召回 → 注入 → 抽取

记忆系统在一次会话中的完整参与过程。

```
  ┌─ 会话开始 ────────────────────────────────────────────────────────────────┐
  │                                                                            │
  │   用户画像加载 (Layer 2)                                                    │
  │   UserMemory.search(user_id)                                               │
  │   → "该用户偏好简洁回复，常关注利润指标，主要管理华东区"                       │
  │                                     ↓ 注入 system_prompt 尾部               │
  │                                                                            │
  │   场景记忆·时间召回 (Layer 1)                                               │
  │   Memory.search(user_id, filters={scene_id, 近3天})                         │
  │   → 该场景下近期交互的关键事实                                                │
  │                                     ↓ 注入 system_prompt 尾部               │
  │                                                                            │
  └──────────────────────────────────────────────────────────────────────────────┘

  ┌─ 每轮对话 ────────────────────────────────────────────────────────────────┐
  │                                                                            │
  │   场景记忆·语义召回 (Layer 1)                                               │
  │   Memory.search(query=用户消息, user_id, filters={scene_id})                │
  │   → 与当前问题语义相关的历史事实                                               │
  │                                     ↓ 注入当轮上下文                         │
  │                                                                            │
  │   对话上下文 (Checkpointer)                                                 │
  │   thread_id = session_id → 自动加载/保存完整消息历史                          │
  │   超阈值时自动 LLM 摘要压缩                                                  │
  │                                                                            │
  └──────────────────────────────────────────────────────────────────────────────┘

  ┌─ 会话结束 (异步) ─────────────────────────────────────────────────────────┐
  │                                                                            │
  │   场景记忆抽取 (Layer 1)                                                    │
  │   Memory.add(messages, user_id, infer=True, metadata={scene_id})            │
  │   → PowerMem 内部自动：事实抽取 → 去重 → 冲突解决 → 遗忘曲线 → 存储          │
  │                                                                            │
  │   用户画像更新 (Layer 2)                                                    │
  │   UserMemory.add(messages, user_id)                                         │
  │   → 自动提取用户偏好/关注点增量，更新全局画像                                  │
  │                                                                            │
  │   不影响对话响应（异步执行，失败仅记日志）                                     │
  │                                                                            │
  └──────────────────────────────────────────────────────────────────────────────┘
```

### 4.5 参数透传流：认证与身份在全链路中的传递

```
  三方系统                      后端框架                       MCP Server
  ────────                    ────────                       ──────────

  ┌─────────┐               ┌──────────────┐              ┌──────────────┐
  │         │   user_id     │              │  user_id     │              │
  │ 场景一   │   extra_params│  request     │  extra_params│  MCP 层      │
  │ 实时Chat │   token      │  _context    │  token(有值) │              │
  │         ├──────────────→│  (ContextVar)├─────────────→│ 有token:     │
  │         │               │              │              │  直接鉴权    │
  └─────────┘               └──────────────┘              │              │
                                                           │ 无token:     │
  ┌─────────┐               ┌──────────────┐              │  用user_id + │
  │         │   creator_id  │              │  user_id     │  extra_params│
  │ 场景二   │   creator_    │  request     │  extra_params│  自行获取    │
  │ 定时任务 │   extra_params│  _context    │  token=None  │  token       │
  │ (无token)├──────────────→│  (ContextVar)├─────────────→│              │
  │         │  从DB恢复      │              │              │              │
  └─────────┘               └──────────────┘              └──────────────┘

  设计原因：定时任务被唤起时无实时 chat 场景，不存在有效 token，
           预存的 token 也会过期，因此将 token 获取下沉到 MCP 层。
```

## 五、热更新机制

```
  ┌──────────────────────────────────────────────────────────────────────────┐
  │                            热更新原理                                    │
  │                                                                          │
  │  管理员修改配置                                                           │
  │       │                                                                  │
  │       ▼                                                                  │
  │  管理页面 → REST API → Service → DB 写入                                 │
  │                                    │                                     │
  │                                    │ ← 运行中的对话不受影响               │
  │                                    │   (已组装的 Agent 实例不变)           │
  │                                    │                                     │
  │  下次用户对话到达 ──────────────────┘                                     │
  │       │                                                                  │
  │       ▼                                                                  │
  │  chat_service → assembler.assemble(scene_id)                             │
  │       │                                                                  │
  │       ▼                                                                  │
  │  从 DB 读取最新配置 → 用新配置组装 Agent → 执行对话                        │
  │                                                                          │
  │  核心：Agent 实例是请求级别的，不是单例。每次请求重新组装。                  │
  │        只有 LLM 实例做缓存（配置变化时 key 变化，自动重建）。               │
  │                                                                          │
  └──────────────────────────────────────────────────────────────────────────┘

  各资源热更新行为：

  ┌──────────────────────┬──────────────────┬──────────────────────────────┐
  │ 修改内容              │ 生效时机          │ 机制                          │
  ├──────────────────────┼──────────────────┼──────────────────────────────┤
  │ Agent (Prompt/模型)   │ 下次对话请求      │ assembler 从 DB 读取          │
  │ Scene 编排 (增减Agent) │ 下次对话请求      │ assembler 从 DB 读取          │
  │ 工具绑定 (增减工具)    │ 下次对话请求      │ assembler 从 DB 读取          │
  │ Skill 内容            │ 下次对话请求      │ Skill 索引重建 + 内容实时查询   │
  │ MCP 工具列表          │ 手动刷新后的下次请求│ mcp_service 更新 DB → 读取    │
  │ 定时任务 (新增/暂停)   │ 立即             │ scheduler_service.refresh()   │
  │ 内置工具              │ 重启后            │ 代码级别，需部署               │
  └──────────────────────┴──────────────────┴──────────────────────────────┘
```

  │                                                                            │
  │  ┌─ 定时任务 ───────────────────────────────────────────────────────────┐  │

  │  │  PowerMem 自动创建的表 (向量存储 + 全文索引)                           │  │
  │  │  · Memory 表     — 场景记忆 (metadata 含 scene_id 用于隔离)           │  │
  │  │  · UserMemory 表 — 用户画像 (按 user_id 全局共享)                     │  │

  │  特性: 向量搜索 + 全文检索 + 关系型 CRUD 统一在一个引擎                      │
  │  约束: 所有表不使用 FK 约束，引用完整性由应用层保证                            │
  │                                                                            │
  └────────────────────────────────────────────────────────────────────────────┘
```

**核心表关系：**

| 表 | 用途 | 关键关联 |
|---|------|---------|
| **agents** | Agent 定义（名称/模型/Prompt） | 被 scenes 引用为主Agent或子Agent |
| **skills** | Skill 定义（指令/脚本/触发方式） | 通过 agent_skill_bindings 绑定到 Scene 中的 Agent |
| **scenes** | Scene 编排（main_agent_id + sub_agent_ids[]） | 核心枢纽，关联 Agent + 绑定表 |
| **agent_tool_bindings** | 工具绑定（Scene × Agent × Tool） | 连接 scenes ↔ agents ↔ mcp_tools/内置工具 |
| **agent_skill_bindings** | 技能绑定（Scene × Agent × Skill） | 连接 scenes ↔ agents ↔ skills |
| **mcp_servers / mcp_tools** | MCP Server 连接配置与工具 Schema 缓存 | mcp_tools 被 agent_tool_bindings 引用 |
| **sessions** | 会话映射 | user_id ↔ session_id ↔ scene_id |
| **scheduled_tasks** | 定时任务定义 | 关联 scene_id，含 creator 身份信息 |
| **task_execution_logs** | 定时任务执行日志 | 关联 task_id |
